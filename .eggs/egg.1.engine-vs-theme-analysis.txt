üìã EXECUTIVE SUMMARY
-------------------
The Boxy CLI currently has a mixed theme command namespace that needs clear separation
between Theme Engine configuration (file/system-level operations) and Individual Theme
usage (style/presentation operations).

üîç CURRENT THEME COMMAND LANDSCAPE
----------------------------------
All theme-related commands are currently under `boxy theme <action>`:

ENGINE OPERATIONS (Manage YAML theme files):
--------------------------------------------
1. list        - Show available themes
2. create      - Create new theme configuration
3. import      - Import theme from external file
4. export      - Export a theme to YAML
5. init        - Initialize local theme template
6. hierarchy   - Show theme loading sources
7. edit        - Modify existing theme configuration

THEME OPERATIONS (Individual theme interactions):
------------------------------------------------
1. show        - Display theme details
2. dryrun      - Test theme rendering
3. help        - Theme-related help

üéØ RECOMMENDED NAMESPACE SEPARATION
-----------------------------------
Proposed CLI Structure:

1. Theme Engine Commands: `boxy theme-engine <action>`
   - theme-engine list
   - theme-engine create
   - theme-engine import
   - theme-engine export
   - theme-engine init
   - theme-engine hierarchy
   - theme-engine edit

2. Theme Preview/Test Commands: `boxy theme-preview <action>`
   - theme-preview show
   - theme-preview dryrun
   - theme-preview help

3. Existing CLI Theme Usage: Remains unchanged
   - `boxy --theme <theme_name>`

üí° RATIONALE
------------
- Clear separation of concerns
- More intuitive command structure
- Prevents namespace pollution
- Makes each namespace's purpose explicit
